# SOUL.md — JSG Holdco Charter

> The test an idea must pass to become a JSG asset — *upstream* of the advisory
> board and kill-criteria, which judge a project already in flight. This decides
> whether an idea belongs in the portfolio at all.
> **Not a mission-vibes page.** If a line here isn't a filter or a fact, cut it.

---

## Identity

JSG is an AI-native holding company. **JSG Estate Liquidators** (Denver-metro estate liquidation, auction house, e-commerce consignment) is the cash-flowing parent. Under it, standalone AI-native businesses are incubated as sellable assets. The operator drives strategy and builds; execution partners run specific accounts. **The moat is the accumulated context — repos, CLAUDE.md files, Mission Control — not any single tool, model, or agent.**

## The Target

Both are wins. No talking down the dream; no inflating the data.

- **Mars (the audacious version):** An empire. The estate-sale / reselling / e-commerce engine inspired VZT; VZT — and the one-off tools built along the way — proved the thesis *and* the operator's own capability ("Dang, I can do this"). Mars is JSG holding **multiple flagship, Mars-driven assets** *while* continuously **developing a pipeline of assets sold at significant multiples**. Both at once: keepers that compound, and a factory that flips.
- **Honest floor (still a win):** JSG's core liquidation / auction / consignment cash flow steady; VZT live and multi-tenant with paying tenants; 1–2 additional assets shipped and generating revenue. Proof the factory works, even while the empire is still early.

## Every JSG Asset Must Be

1. **AI-native — leverage, not headcount.** If scaling it means hiring, it's the wrong build.
2. **Sellable from day one.** Clean separation of concerns, documented systems, transferable infrastructure, no founder-dependent magic. Built to hand to a buyer.
3. **Context-portable.** Any agent can cold-start from the repo's CLAUDE.md and be useful. Tool-agnostic; nothing married to a vendor.
4. **On a credible path to revenue.** A plausible time-to-revenue exists *before* the build starts.

## What We Will NOT Build

The filter that kills shiny objects before they eat a build window.

- **Founder-dependent anything.** If it only works because David runs it, it can't be sold.
- **Fragile node-based glue** (Make / Zapier / n8n) as a core dependency.
- **Tools we'd marry** — no core dependency on a vendor we can't swap.
- **Non-transferable one-offs** with no exit and no leverage.
- **Single-client service businesses.** Revenue dependent on one client is fragile and effectively non-sellable as an asset.
- **Regulated verticals (health / finance / legal) without a partner** absorbing the compliance burden.

## Exit Thesis

- **Asset-level:** Flippa or Empire Flippers at revenue multiples, or strategic acquisition. Every architectural choice serves transferability.
- **Holdco-level:** **JSG is a permanent cash-flowing parent — the factory.** It spins out and sells individual assets; it is not itself for sale. This is *why* clean per-asset separability is non-negotiable: each asset must be sellable without unwinding the parent.

## How This Charter Is Used

- **Project intake gate:** a new idea is tested against *Every JSG Asset Must Be* + *What We Will NOT Build* **before** it enters the build pipeline. Fail → it's not a JSG project. Kill it, or explicitly log it as personal/out-of-scope (like the video optimizer).
- **Order of gates:** SOUL charter (*does it belong?*) → advisory board (*should we build it now?*) → kill-criteria (*is it still viable in flight?*).
- Lives in the OS repo + the vault; agents read it at intake.

## What This File Is NOT

Not values, not vibes, not aspiration that doesn't filter a decision. If reality changes — the portfolio reshuffles, the thesis sharpens — update this file. A stale charter is worse than none.
